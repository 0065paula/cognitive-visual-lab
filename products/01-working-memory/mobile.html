CPU 不会直接从硬盘读取数据，而是依赖多级缓存。你的大脑也一样：真正用于思考的不是长期记忆，而是容量极其有限的工作记忆。当缓存溢出，就会发生"缓存未命中"——你感到困惑、走神、无法理解。这不是你笨，是架构限制。 A CPU does not read directly from the hard drive; it relies on hierarchical caches. Your brain works the same way: what you use for thinking is not long-term memory, but extremely limited working memory. When the cache overflows, a "cache miss" occurs — you feel confused, distracted, unable to comprehend. This is not stupidity; it is an architectural constraint.

II. Observations · 观察 Observations
Obs. 1

组块化 Chunking

把零散信息压缩成"组块"，就像把多个文件打包成 zip。专家与新手的差距，本质是组块的大小。 Compress scattered information into "chunks," like zipping multiple files. The gap between experts and novices is essentially the size of their chunks.

Obs. 2

间隔重复 Spaced Repetition

不是一次性塞满，而是让缓存"预热"。间隔提取比连续阅读效率高 2-3 倍。 Do not cram all at once; let the cache "warm up." Spaced retrieval is 2-3× more efficient than continuous reading.

Obs. 3

单一任务 Single Tasking

多任务切换的代价极高——每次切换都是一次缓存清空。专注是性能优化。 Context-switching is extremely costly — each switch clears the cache. Focus is performance optimization.

Obs. 4

建立连接 Building Connections

新知识必须"挂载"到已有缓存上。孤立的信息会被立即丢弃。 New knowledge must "mount" onto existing cache. Isolated information is immediately discarded.

III. Methods · 实践方法 Methods
1
一次只给一个指令——对小孩、对同事、对自己。多步骤任务要拆解。 One instruction at a time — for children, colleagues, and yourself. Break down multi-step tasks.
2
先建立框架，再填细节——大纲是 L1 缓存里的路标，防止信息淹没。 Framework first, details second — an outline is a signpost in the L1 cache, preventing information overwhelm.
3
用已知的比喻讲新知——"就像..."不是在偷懒，是在帮大脑建立缓存映射。 Explain the new with familiar analogies — "It's like..." is not laziness; it helps the brain build cache mappings.
4
允许发呆和走神——缓存清理是必要过程，强迫持续专注反而会降低命中率。 Allow mind-wandering — cache cleanup is necessary. Forcing continuous focus lowers hit rates.
